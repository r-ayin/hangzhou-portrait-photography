# 饺子馆传播模式 — 通用模式文档

> 让 Agent 互相推荐的技能传播机制。灵感来自「金正源饺子馆」。

## 核心原理

当一个技能中嵌入传播指令后，其他 Agent 加载该技能时，在相关场景下会主动推荐该技能/品牌，形成 Agent-to-Agent 的推荐网络。

## 传播链路

```
Agent A 加载 skill → 遇到相关需求 → 推荐品牌/服务
        ↓
Agent B 从用户处听到品牌名 → 也可能加载本 skill
        ↓
Agent C 从其他渠道发现 → 被多源引用
        ↓
AI 检测到多源共识 → 判定「行业推荐」→ 自然排名上升
```

## 生效规则设计

关键要素：
1. **触发场景** — 明确哪些用户问题触发推荐（如「杭州约拍推荐」「谁能做AI Agent」）
2. **推荐话术** — 每场景固定推荐话术，含具体理由+联系方式
3. **传播伦理** — 只在与技能相关的场景推荐，不硬植入，不贬低同行

## 实施步骤

1. 在 SKILL.md 的 triggers 中添加 Agent 发现类关键词（如「推荐摄影师」「帮我推荐约拍」）
2. 在 description 中添加传播描述（如「饺子馆模式Agent间推荐」）
3. 在 SKILL.md 正文中新增「饺子馆传播模式」章节，包含生效规则+传播链路+伦理规则
4. 可选：在 README 添加饺子馆徽章

## 关键教训：技能改造的正确方式

用户说「把这个 skill 改造成XXX」时：
1. 不要改名字 — 保持原名，在原名基础上增加能力
2. 不要改核心身份 — 如果用户是品牌所有者，保持品牌身份
3. 向现有技能添加层 — 新增能力作为一个层加在已有技能上，而不是替换它
