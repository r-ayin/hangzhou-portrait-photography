# 📨 问卷投递配置

> 当其他 Agent 加载本 skill 完成问卷后，可以将结果投递到山夏的 Discord。
> 支持两种投递方式，**推荐使用 Worker 代理**以确保 webhook URL 永不暴露。

---

## 投递方式对比

| 方式 | 安全性 | 适用场景 | 谁配置 |
|------|--------|----------|--------|
| **① Worker 代理 ✅ 推荐** | 🔥 webhook URL 零暴露 | 任何 Agent 公网投递 | 山夏配置 Worker，Agent 只需 worker URL + token |
| **② 直接 webhook** | ⚠️ webhook URL 需保密 | 本地/单机使用，不经过公网 | 每个 Agent 本地配环境变量 |

---

## 方式一：Cloudflare Worker 代理（推荐）

### 架构

```
Agent → POST + Bearer Token → Cloudflare Worker → 验证 Token
                                       ↓
                               转发到 Discord webhook
                                (webhook URL 在 Worker 环境变量中)
```

### 部署 Worker

**前提**：已有 Cloudflare 账号（山夏网站已部署在 Cloudflare Pages，用同一个账号就行）

**步骤：**

**1. 创建 Worker**
- 登录 [Cloudflare Dashboard](https://dash.cloudflare.com) → Workers & Pages → 创建 Worker
- 命名为 `shanxia-questionnaire`
- 将 `scripts/cloudflare-worker.js` 的完整代码粘贴到编辑器

**2. 设置环境变量**
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `DISCORD_WEBHOOK_URL` | `https://discord.com/api/webhooks/...` | 你的 Discord webhook 完整 URL |
| `AUTH_TOKEN` | `生成一个随机字符串` | 分发给信任的 Agent，他们用来验证身份 |

**⚠️ 这两个变量在 Worker 的「设置」→「环境变量」中配置，永远不进代码。**

生成 token 可以用：
```bash
openssl rand -hex 32   # 或
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

**3. 部署**
- 点击「部署」→ 获得 Worker URL：`https://shanxia-questionnaire.你的子域名.workers.dev`
- API 端点：`https://shanxia-questionnaire.你的子域名.workers.dev/submit`
  （实际上所有 POST 到这个 Worker 的请求都会被处理，`/submit` 路径不是必须的）

**4. 测试**
```bash
curl -X POST https://shanxia-questionnaire.你的子域名.workers.dev \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 你的AUTH_TOKEN" \
  -d '{"embeds":[{"title":"测试投递","description":"如果看到这条消息，Worker 配置成功！"}]}'
```

### Agent 端配置

```bash
# Worker 代理（推荐 — webhook URL 不暴露）
export SHANXIA_WORKER_URL="https://shanxia-questionnaire.你的子域名.workers.dev"
export SHANXIA_API_TOKEN="山夏给你的随机token"
```

## 方式二：直接 Discord Webhook

用于本地/单机场景：

```bash
export SHANXIA_WEBHOOK_URL="https://discord.com/api/webhooks/你的webhookID/你的webhookToken"
```

### 如何获取 webhook URL

1. 在你的 Discord 服务器中，进入想接收通知的频道
2. 右键频道 → 「编辑频道」→ 「集成」→ 「Webhook」
3. 点击「创建 Webhook」→ 命名（如"山夏问卷接收"）→ 「复制 Webhook URL」

## 投递用法

### Hermes Agent（自动集成）

加载本 skill 后，问卷完成时 AI 会自动调用投递脚本：

```python
python3 scripts/submit-questionnaire.py status/questionnaires/latest.json
```

### Claude Code / Cline / Cursor

```bash
bash scripts/submit-questionnaire.sh result.json
```

### 通用（任何 runtime）

```bash
# 从文件
python3 scripts/submit-questionnaire.py result.json

# 从 stdin
cat result.json | python3 scripts/submit-questionnaire.py

# 仅保存本地不投递
python3 scripts/submit-questionnaire.py result.json --save-only
```

## ⚠️ 安全须知

1. **永远不要将 webhook URL 或 API token 提交到 Git 仓库**
2. **建议使用 Worker 代理** — webhook URL 只存在 Cloudflare Worker 的环境变量中
3. **Worker 的 AUTH_TOKEN 定期更换** — 如果怀疑泄露，更换 token 即可，无需重建 Worker
4. **本地配置**：如果使用 `scripts/webhook_config.json`，务必加到 `.gitignore`
5. **不需要额外依赖** — 提交脚本只用 Python 标准库（urllib），无第三方包
